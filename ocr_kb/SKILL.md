# 论文 OCR 与知识库生成工作流 (Multimodal OCR & KB Pipeline)

## Overview
此技能充分利用多模态大模型（如 Gemini、Claude）的原生视觉能力。大模型通过直接读取文档的单页图片，不仅免去了复杂的本地模型部署或外部 API 调用，而且能一次性完成文本识别、公式 LaTeX 提取、语义错误核对以及文档层级重构。

## 0. 架构底线 (Architecture Constraints)
*   **System Source of Truth (系统真值)**：原始提取的高清图片（`resources/pages/*.png`）是唯一的真值。生成的 `.md` 知识库仅仅是真值的一层结构化投影（Projection）。
*   **Module Boundaries (模块边界)**：本技能（`ocr_kb`）**只负责**读取图片并生成结构化的 Markdown 知识库。它**绝不负责**生成最终的 `.docx` 排版文件（该任务属于 `docx` 技能）。
*   **Failure States (失败态与恢复)**：如果遇到图片模糊无法识别、公式被截断、或语言无法解析等失败态，**禁止脑补或省略**，必须在生成的文本中显式输出 `[HARD FAILURE: 无法识别文本，请提供更高清原图]` 并暂停。

## 1. 准备工作：文档转图片
在使用多模态图片识别功能前，必须确保原始 PDF 文稿已被切分为按页的图片。
- 确保所有的图片保存在 `resources/pages/`（或相关输入目录）。
- 运行配套脚本将 `.pdf` 转换为高分辨率图片：

```bash
# 请将 input.pdf 替换为实际文件名
python ocr_kb/scripts/page_to_image.py draft.pdf resources/pages/
```
*(注：如果原始文件是 Word (.docx)，可使用 `docx` 技能中的方法先转为 PDF。)*

## 2. 原生增量分块引擎 (Chunking & Checkpoint Engine)
> [!IMPORTANT] 防截断强制协议
> 这是防止 Agent 在处理长文档（20-50页）时因上下文超限而擅自“只翻译前几页就交差”的**唯一合法处理机制**。

当用户触发 `ocr_kb` 技能批量处理页面时，你**必须严格执行以下三步：**
1.  **盘点摸底**：使用 `list_dir` 获取 `resources/pages/` 下的图片总数。
2.  **强制写计划**：通过写入 `task.md` 文件，创建**每批次最多 5 张图片**的核对清单（例如 `[ ] OCR Pages 1-5`, `[ ] OCR Pages 6-10`）。
3.  **自动接力跑 (Relay Execution)**：
    *   调用 `task_boundary` 启动第一个批次（如处理 1-5 页）。
    *   通过 `view_file` 读取这 5 张图片，将萃取的文本按规则追加进 `outputs/knowledge_base_reformatted.md`。
    *   在 `task.md` 中打钩 `[x]`。
    *   **立刻连续调用下一个 `task_boundary` 开启下一个 5 页的批次（如 6-10 页），绝不允许向用户发消息中断流程！**
    *   循环往复接力，直到最后一张图处理完毕，才可以 `notify_user` 通知用户。

## 3. 页面处理视角核心 (Agent Vision Prompt)

> **对批次内的每一页图片，你在心里执行以下三个维度的处理，并在最后整合输出：**

### 一、 核心 OCR 与公式转 LaTeX
1.  **精准提取**：全文提取图片上的所有文字。保持原有的段落连续性。
2.  **公式处理**：识别出图片中所有的数学公式（甚至是题注中包含的单字符）。将它们**强制转换为标准的 LaTeX 语法代码**（使用 `$` 表示行内，`$$` 表示块级）。

### 二、 快捷语义分割与原则性错误核对
1.  **原则性核对**：凭借你的知识储备，迅速留意是否存在明显的常识性错误、上下文断层、或逻辑矛盾的段落。
    *   若发现问题，立刻在文本插入警告：`> [!WARNING] 语义核对提示：此处定义与上下文似乎有矛盾，请检查。`

### 三、 序列化层级重排 (Hierarchical Formatting)
1.  **自动规整**：根据文章的视觉字号排版（大标题、小标题、首行缩进等）及语义深度，梳理为严谨的序号排列：`1`, `1.1`, `1.1.1`。
2.  即使原稿使用的是字母或中文编号体系，也**统一映射并强制转化为** `1, 1.x, 1.x.x` 的通用知识库格式。

### 四、 失败态处理 (Failure Handling)
1.  如果图片内容极其模糊导致无法辨认，**不要尝试胡编乱造**。请在该位置插入严格的失败态标记：`> [!CAUTION] [HARD FAILURE: 无法识别文本，请提供更高清原图]`。

## 4. 执行范例 (Execution Walkthrough)
1. 用户：“提取这个长 PDF 建立知识库”
2. Agent 运行切图脚本得到 22 张图。
3. Agent 遵守防截断协议，写出 5 批任务清单到 `task.md` (`1-5`, `6-10`, `11-15`, `16-20`, `21-22`)。
4. Agent 无缝接力，开一个 `task_boundary` 读 5 页写盘，勾选清单，再开一个接着读。
5. 直到全本 OCR 提取结束，向用户汇报，并在用户授意下，交接给 `docx` 技能进行最终排版与公式（OMML）复原。
