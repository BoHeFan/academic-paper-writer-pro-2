# 论文 OCR 与知识库生成工作流 (Multimodal OCR & KB Pipeline)

## Overview
此技能充分利用多模态大模型（如 Gemini、Claude）的原生视觉能力。大模型通过直接读取文档的单页图片，不仅免去了复杂的本地模型部署或外部 API 调用，而且能一次性完成文本识别、公式 LaTeX 提取、语义错误核对以及文档层级重构。

## 0. 架构底线 (Architecture Constraints)
*   **System Source of Truth (系统真值)**：原始提取的高清图片（`resources/pages/*.png`）是唯一的真值。生成的 `.md` 知识库仅仅是真值的一层结构化投影（Projection）。
*   **Module Boundaries (模块边界)**：本技能（`ocr_kb`）**只负责**读取图片并生成结构化的 Markdown 知识库。它**绝不负责**生成最终的 `.docx` 排版文件（该任务属于 `docx` 技能）。
*   **Failure States (失败态与恢复)**：如果遇到图片模糊无法识别、公式被截断、或语言无法解析等失败态，**禁止脑补或省略**，必须在生成的文本中显式输出 `[HARD FAILURE: 无法识别文本，请提供更高清原图]` 并暂停。

## 1. 准备工作：文档转图片
在使用多模态图片识别功能前，必须确保原始 PDF 文稿已被切分为按页的图片。
- 确保所有的图片保存在 `resources/pages/`（或相关输入目录）。
- 运行配套脚本将 `.pdf` 转换为高分辨率图片：

```bash
# 请将 input.pdf 替换为实际文件名
python ocr_kb/scripts/page_to_image.py draft.pdf resources/pages/
```
*(注：如果原始文件是 Word (.docx)，可使用 `docx` 技能中的方法先转为 PDF。)*

## 2. 核心工作流：多模态识别与重构 (Agent Prompt)

> **给 Agent (你自己) 的操作指令**：
> 作为多模态大模型，当你处理这些图片时，你需要**逐一查看**转换好的一页页图片，利用你强大的视觉能力，针对每一页输出如下规格的内容。最后将它们汇总保存为 `.md` 知识库文件。

对于**每一页**图片，你需要执行并输出以下三个维度的处理：

### 一、 核心 OCR 与公式转 LaTeX
1.  **精准提取**：全文提取图片上的所有文字。保持原有的段落连续性。
2.  **公式处理**：识别出图片中所有的数学公式（无论是行内公式、独立居中公式块、带序号公式，还是题注中包含的单字符）。将它们**强制转换为标准的 LaTeX 语法代码**（使用 `$` 表示行内公式，`$$` 表示独立块公式）。
    *   *要求*：对微小字符、上下标、特殊希腊字母进行细致还原。

### 二、 快捷语义分割与原则性错误核对
1.  **原则性核对**：在提取过程中，凭借你的知识储备，迅速留意是否存在明显的常识性错误、上下文断层、或逻辑矛盾的段落。
    *   **若发现潜在问题**：请在输出文本中立刻插入显眼的标签（如：`> [!WARNING] 语义核对提示：此处定义与上下文似乎有矛盾，请检查。`），以便于后续人工干预和审查。

### 三、 序列化层级重排 (Hierarchical Formatting)
1.  **自动规整**：根据文章的视觉字号排版（大标题、小标题、首行缩进等）及语义深度，将提取后的文本重新梳理为严谨的序号排列：`1`, `1.1`, `1.1.1`。
2.  即使原稿使用的是其他编号体系（如 "A. / B. / C.", "I. / II.", 或中文字符 "一、 / 二、"），也**统一映射并强制转化为** `1, 1.x, 1.x.x` 的通用知识库格式。

### 四、 失败态处理 (Failure Handling)
1.  如果图片内容极其模糊导致无法辨认，**不要尝试胡编乱造**。请在该位置严格输出提示：`> [!CAUTION] 识别失败：图片分辨率过低或内容损坏，请人工核查该页。`

## 3. 输出与合成
处理完所有页面后，将拼接的内容保存到专门的知识库格式文件中，例如 `outputs/knowledge_base_reformatted.md`。

## 执行范例
当用户要求：“将这篇PDF转化为带编号规范的格式化知识库文本，并注意公式转Latex” 时，Agent 应该：
1. 确认 PDF 文件路径。
2. 执行 `python ocr_kb/scripts/page_to_image.py input.pdf resources/pages/` 得到每页的 PNG。
3. 循环调用自己的图片理解能力，传入 prompt 提取 `page_001.png`，随后附加到 `.md` 中。
4. 重复该过程直到最后一页。
